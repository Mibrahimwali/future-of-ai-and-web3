# Contributing to future-of-ai-and-web3

Thank you for helping build a credible public knowledge library.

## Content standards

- 800–1500 words per article
- Required sections: Introduction, Problem Statement, Solution, Architecture or Framework, Benefits, Challenges, Future Opportunities, Conclusion
- One primary keyword and three secondary keywords per article
- Educational, actionable, human-readable writing — no filler

## Pull requests

1. Fork the repository
2. Add content under `research/YYYY/MM/slug.md`
3. Update `INDEX.md` with a link to your article
4. Open a PR with a descriptive title matching our commit style

## Code of conduct

Be respectful and constructive. This library represents long-term professional credibility.
